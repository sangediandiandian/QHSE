# 石油炼化企业QHSE系统原型

## 需求分析与概要设计文档

**文档版本：** V1.0 **建设阶段：** 原型设计阶段 **开发方式：** Vibe Coding + Mock数据 **适用对象：** 炼油、化工、储运、公用工程等石油炼化企业 **系统定位：** QHSE综合管理、风险监测预警与应急联动平台

# 一、项目背景

石油炼化企业生产装置密集，介质具有易燃、易爆、有毒、有害、高温、高压等特点。企业通常已经建设MES、GDS、VOC在线监测、视频监控、人员定位、消防报警、融合通信等专业系统，但各系统之间相对独立，存在以下问题：

1. VOC、可燃气体、有毒气体和生产工艺数据分散在不同系统。
2. 单一系统告警无法结合生产状态进行综合研判。
3. 告警发生后，依赖值班人员人工查找预案、电话通知。
4. 应急通知渠道分散，无法快速确认人员是否收到信息。
5. 事件处置过程缺少任务跟踪、升级、反馈和复盘。
6. 管理人员无法在一个界面查看风险、告警和处置状态。
7. 告警数据与隐患、设备、作业票、承包商及历史事件关联不足。

本项目拟建设一套QHSE系统原型，将QHSE管理业务与VOC、GDS、MES等系统数据进行关联，形成统一的风险监测、预警研判、应急响应和闭环处置能力。

# 二、建设依据

原型设计可参考以下管理体系及工程标准：

- Q/SY 08002.1—2022《健康、安全与环境管理体系 第1部分：规范》
- GB/T 24001—2016《环境管理体系 要求及使用指南》
- GB/T 45001—2020《职业健康安全管理体系 要求及使用指南》
- GB/T 50493—2019《石油化工可燃气体和有毒气体检测报警设计标准》
- 企业现有QHSE管理制度
- 企业应急预案、专项预案和现场处置方案
- 企业VOC、GDS、MES和融合通信系统接口规范

GB/T 24001—2016和GB/T 45001—2020目前在国家标准平台标记为现行标准；GB/T 50493—2019自2020年1月1日起实施，覆盖石油化工可燃气体和有毒气体检测点、报警系统及安装设计等内容。([国家标准全文公开系统](https://openstd.samr.gov.cn/bzgk/std/newGbInfo?hcno=61011DD84D5D3AB7F5BCFEA911460D3D&utm_source=chatgpt.com))

原型阶段主要体现管理逻辑、系统交互和业务闭环，不替代企业现有安全仪表系统、联锁系统、GDS控制器、DCS或消防控制系统。

# 三、建设目标

## 3.1 总体目标

建设一个面向石油炼化企业的QHSE综合管理原型，实现：

1. QHSE基础业务数字化。
2. VOC、GDS、MES数据统一展示。
3. 多源数据综合预警。
4. 预警事件与应急预案自动匹配。
5. 通过融合通信系统快速发送语音告警。
6. 应急任务自动生成、分派和跟踪。
7. 告警、处置、确认、关闭和复盘全过程留痕。
8. 通过大屏、地图和驾驶舱直观展示企业风险状态。

## 3.2 原型演示目标

原型应重点演示以下场景：

### 场景一：GDS可燃气体超限

某装置可燃气体探测器浓度达到一级报警值，系统自动获取：

- 探测器位置；
- 所属装置和区域；
- 检测介质；
- 当前浓度及变化趋势；
- 周边生产设备；
- MES生产状态；
- 附近作业票；
- 周边人员和应急资源。

系统生成预警事件，匹配“可燃气体泄漏现场处置方案”，并通过融合通信系统向岗位人员、班长、调度、安全管理人员发送语音告警。

### 场景二：VOC排放异常

某排口非甲烷总烃连续超限，系统关联：

- 排口信息；
- VOC实时浓度；
- 排放流量；
- 生产装置运行负荷；
- 物料切换记录；
- 治理设施运行状态；
- 风机、阀门和RTO/RCO等设施状态；
- 环保指标限值。

系统判断为环境异常事件，通知环保管理人员、装置负责人和生产调度，并生成核查任务。

### 场景三：工艺参数与GDS联合预警

MES显示某设备压力、温度或流量异常，同时附近GDS探测器浓度持续上升。

系统将单点告警升级为“疑似工艺介质泄漏”，提高事件等级，自动启动专项应急流程，并扩大语音通知范围。

### 场景四：告警无人确认自动升级

系统发送语音告警后，在设定时间内未收到责任人确认，自动：

1. 再次呼叫责任人；
2. 通知上一级负责人；
3. 通知生产调度和应急指挥人员；
4. 在应急大屏显示红色待响应状态；
5. 记录每次呼叫时间、人员和结果。

# 7.2 风险分级管控

## 7.2.1 风险单元管理

风险单元层级：

企业 → 厂区 → 运行部 → 装置 → 单元 → 设备 → 测点。

风险单元信息：

- 风险单元编号；
- 风险单元名称；
- 所属区域；
- 责任单位；
- 责任人；
- 经纬度或平面坐标；
- 涉及介质；
- 设备设施；
- 事故类型；
- 当前风险等级；
- 关联GDS测点；
- 关联VOC排口；
- 关联MES设备；
- 关联应急预案。

## 7.2.2 风险辨识

支持记录：

- 危险源；
- 环境因素；
- 可能发生的事件；
- 发生可能性；
- 后果严重程度；
- 风险等级；
- 管控措施；
- 责任岗位；
- 检查频次；
- 应急措施。

# 九、综合预警中心

# 9.1 预警来源

- GDS系统；
- VOC系统；
- MES系统；
- 人员上报；
- 隐患系统；
- 作业票系统；

- 视频AI；
- 气象系统；
- 消防系统。

# 9.2 预警等级

建议分为四级：

| 等级  | 颜色 | 含义       | 处置要求             |
| ----- | ---- | ---------- | -------------------- |
| IV级  | 蓝色 | 提示性异常 | 岗位关注并核查       |
| III级 | 黄色 | 一般预警   | 班组和装置负责人处置 |
| II级  | 橙色 | 较大预警   | 调度及QHSE部门介入   |
| I级   | 红色 | 重大预警   | 启动应急响应         |

最终等级应允许企业根据内部制度配置。

# 9.3 预警规则示例

## 规则一：GDS一级报警

条件：

- 当前值达到一级阈值；
- 持续时间超过10秒。

动作：

- 生成黄色预警；
- 通知岗位人员；
- 调取附近视频；
- 展示现场处置卡；
- 要求2分钟内确认。

## 规则二：GDS二级报警

条件：

- 当前值达到二级阈值。

动作：

- 生成橙色或红色预警；
- 自动匹配泄漏应急预案；
- 通知装置负责人、调度和QHSE人员；
- 启动融合通信语音告警；
- 自动生成现场核查任务；
- 显示附近人员和应急资源。

## 规则三：GDS与MES联合预警

条件：

- GDS浓度持续上升；
- 同区域MES压力或流量异常；
- 两类异常时间间隔小于5分钟。

动作：

- 判断为疑似工艺介质泄漏；
- 将事件等级提升一级；
- 扩大告警范围；
- 通知生产调度；
- 推荐执行降负荷、隔离或停车等人工决策项。

系统只能提供处置建议，不直接控制生产设备。

## 规则四：VOC连续超限

条件：

- VOC浓度超过限值；
- 连续超过设定时间；
- 排除仪表故障和数据中断。

动作：

- 生成环保预警；
- 关联治理设施运行状态；
- 关联MES生产负荷；
- 通知环保人员和装置负责人；
- 生成原因核查任务。

## 规则五：VOC与生产切换关联

条件：

- MES记录装置开停车或物料切换；
- VOC浓度在相同时间段快速上升。

动作：

- 标记为“生产操作关联异常”；
- 提示检查密闭排放、回收设施和治理设施；
- 保存生产事件和VOC曲线。

## 规则六：高风险作业区域告警

条件：

- 区域存在有效动火或受限空间作业票；
- 区域GDS出现报警。

动作：

- 立即向作业人、监护人和作业负责人语音告警；
- 作业状态标记为“建议暂停”；
- 要求人员撤离并重新检测；
- 通知属地单位。

# 二十一、原型建设原则

1. 先完成核心演示闭环，再补充管理模块。
2. 原型阶段不追求真实系统全部功能。
3. 所有告警均可通过按钮模拟触发。
4. 所有实时曲线均使用Mock数据动态变化。
5. 不直接控制生产设备。
6. 不将QHSE系统作为SIS、DCS或GDS安全联锁的替代系统。
7. 重点展示“数据融合、智能研判、预案联动、语音通知和闭环处置”。
8. 页面名称和业务术语尽量贴近炼化企业实际。
9. 每一个告警都必须具有来源、责任人、处置过程和关闭依据。
10. 每一次通信都必须能够查看发送、接通、确认和失败记录。

# 二十二、项目核心业务闭环

系统最终应形成以下闭环：

**监测数据进入系统**

↓

**数据校验和设备区域映射**

↓

**预警规则判断**

↓

**生成预警事件**

↓

**关联VOC、GDS、MES、作业票、人员和视频**

↓

**确定事件等级**

↓

**自动匹配应急预案**

↓

**通过融合通信发送语音告警**

↓

**责任人确认**

↓

**启动应急任务**

↓

**现场人员反馈处置结果**

↓

**风险解除和事件关闭**

↓

**转隐患、调查分析和复盘改进**

该闭环是本次QHSE原型最重要的建设内容。

建议实际开发时，不要一开始就建设全部QHSE模块。第一版优先完成四个可演示页面：**综合驾驶舱、监测中心、预警事件详情、应急指挥台**，先把“GDS告警—MES关联—预案启动—语音通知—任务处置”的主场景跑通。
