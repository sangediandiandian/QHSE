## 需要完善的功能和模块

### P0：核心业务补齐

1. 风险分级管控（原型已补齐）
   - 已实现组织级风险单元树、LEC 辨识评估、评估记录和管控措施责任维护。
   - 正式系统待对接风险标准库、服务端审批和全量历史档案。
2. 隐患排查治理（原型已补齐）
   - 已完成 NestJS/Prisma 纵向切片、内存与 PostgreSQL 双仓储、四态状态机、区域数据范围、五类权限、操作审计和乐观锁。
   - 前端 API 模式已直接接入领域接口，不再读取或写入共享 dashboard 隐患缓存；Mock 模式仍可独立演示。
   - 待对接对象存储、移动端拍摄、自动催办和重复隐患聚类。
3. 作业许可管理（原型已补齐）
   - 已完成 NestJS/Prisma 纵向切片、内存与 PostgreSQL 双仓储、五类作业申请、三级分权签署、异人现场确认、暂停/恢复/关闭状态机和乐观锁。
   - 前端 API 模式已直接接入作业许可接口，申请人、签署人和确认人均取当前认证主体。
   - 待接入规则执行引擎自动生成暂停建议，以及 CA 电子签名、GIS 点选、人员定位与移动现场终端。
4. 状态持久化（原型已补齐）
   - 已使用版本化 localStorage 保存告警、隐患、票证、规则、预案和事件生命周期操作，并提供演示数据重置。
   - 风险和隐患域已具备后端数据库模型、并发控制和服务端规则；其余能力域仍需迁移。

### P1：闭环能力增强

1. 预警规则配置（原型已补齐）
   - 已完成服务端草稿、同作用域表达式冲突检测、QHSE/生产负责人异人会签、25%/50%/100% 灰度配置、版本快照、回滚草稿、启停和发布审计。
   - 前端 API 模式已直接接入规则领域接口，草稿和运行版本相互隔离。
   - 待接入实时采样窗口计算、表达式执行、告警去重抑制和灰度分群。
2. 应急事件全生命周期（原型已补齐）
   - 已实现研判、升级、降级、终止、关闭申请、4 小时审批任务、附件证据门槛、审批意见、电子签名、超时催办和操作留痕。
   - 正式系统待对接服务端工作流引擎、CA 签名、对象存储、消息催办与审计归档。
3. 预案管理流程（原型已补齐）
   - 已实现新建、独立草稿、QHSE/生产负责人双人会签、版本记录、内容差异对比、历史回滚、到期提醒、演练计划和复盘记录。
   - 正式系统待接入服务端审批任务、CA 签名、定期演练自动排程和演练附件归档。
4. 资源管理流程
   - 已实现资源入库、批次与有效期、临期/过期识别、FEFO 调拨、过期库存拦截、库存占用、调拨到位、归还、巡检维护和操作轨迹。
   - 待补扫码盘点、维修工单、跨库调拨和服务端库存并发控制。
5. 融合通信真实接口
   - 对接电话、短信、App、广播平台，处理重试、失败回执和号码脱敏。
6. 告警联动证据
   - 已实现事件详情关联监测趋势、工艺参数、同区域作业票、通知人员、证据核验和操作留痕。
   - 待补人员定位、视频快照、附件固化、证据哈希和服务端审计归档。

### P2：平台化与生产化

1. 组织、用户、角色和数据权限管理。
   - 后端基线已完成组织/区域模型、8 类角色、用户授权、会话认证和 IAM 查询接口。
   - 待补管理页面、增删改停用流程、统一身份认证和权限变更审批。
2. 菜单、按钮、数据范围权限和重要操作二次确认。
   - 风险和隐患模块已接入菜单/按钮权限、后端权限点和区域数据范围强制过滤。
   - 待扩展到其他业务菜单，并补重要操作二次确认。
3. 操作日志、登录日志、告警确认日志和预案启动审计。
   - 已完成登录成功/失败、退出、401/403 安全拒绝、风险评估和措施更新审计。
   - 待随业务后端化补齐告警确认、预案启动和通信发送审计。
4. 统计分析、趋势报表、单位绩效和导出能力。
5. 数据字典、编码映射、接口配置和数据质量监控。
6. WebSocket / MQTT 实时数据、断线重连和时间同步。
7. 异常边界、错误恢复、无障碍、移动端和值守大屏长时间运行测试。
8. 后端、数据库、缓存、消息队列、统一认证和部署运维体系。

## 后端建设进度

- 已建立 NestJS 模块化单体骨架、统一响应/错误、请求追踪、Swagger、健康检查和本地开发代理。
- 已完成风险分级后端首个纵向切片：列表/筛选、详情、LEC 评估、管控措施、Prisma 数据模型和乐观并发控制。
- 已完成 IAM 与审计基线：8 类角色、Bearer 会话、权限点、区域数据范围、组织/用户/角色查询和统一审计。
- 已完成隐患治理后端纵向切片和前端 API 接入：状态机、证据门槛、验收、督办、操作轨迹、权限与乐观锁均由服务端强制执行。
- 已完成作业许可后端纵向切片和前端 API 接入：分权审批、双人现场确认、暂停恢复关闭、权限审计和并发控制均由服务端强制执行。
- 已完成通用审批流内核：业务关联、有序节点、角色会签、驳回、撤回、活动流程防重和乐观锁可供预警规则、预案和事件关闭复用。
- 已完成预警规则配置发布后端化和前端 API 接入：草稿、冲突检测、异人双会签、版本发布/回滚、启停、权限审计和并发控制均由服务端强制执行。
- 当前默认使用内存仓储以保证开箱即启，并已提供 PostgreSQL 仓储与种子入口。
- 下一阶段建设预警规则实时执行器，再进入应急事件、预案和资源后端化；详细路线见 [BACKEND_ROADMAP.md](./BACKEND_ROADMAP.md)。

## 建议迭代顺序

1. 风险分级管控 + 隐患治理 + 作业许可，完成 QHSE 管理主业务。
2. 服务端事件审批 + 规则表达式编排，使现有原型具备正式工作流和复杂规则配置能力。
3. 权限审计 + 统计报表 + 基础配置，形成可交付的管理平台。
4. 真实数据和通信接口接入，最后开展性能、安全和生产部署改造。
