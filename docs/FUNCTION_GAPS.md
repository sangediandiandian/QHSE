## 需要完善的功能和模块

### P0：核心业务补齐

1. 风险分级管控（原型已补齐）
   - 已实现组织级风险单元树、LEC 辨识评估、评估记录和管控措施责任维护。
   - API 模式的评估和措施维护已直接写入风险领域接口，评估人取认证主体，页面按服务端返回版本更新并在写后失效驾驶舱缓存。
   - 正式系统待对接风险标准库、服务端审批和全量历史档案。
2. 隐患排查治理（原型已补齐）
   - 已完成 NestJS/Prisma 纵向切片、内存与 PostgreSQL 双仓储、四态状态机、区域数据范围、五类权限、操作审计和乐观锁。
   - 前端 API 模式已直接接入领域接口，不再读取或写入共享 dashboard 隐患缓存；Mock 模式仍可独立演示。
   - 已接入统一对象存储、附件 SHA-256、区域鉴权下载和证据业务绑定。
   - 待补移动端拍摄、自动催办和重复隐患聚类。
3. 作业许可管理（原型已补齐）
   - 已完成 NestJS/Prisma 纵向切片、内存与 PostgreSQL 双仓储、五类作业申请、三级分权签署、异人现场确认、暂停/恢复/关闭状态机和乐观锁。
   - 前端 API 模式已直接接入作业许可接口，申请人、签署人和确认人均取当前认证主体。
   - 已接入规则执行引擎，可在较大/重大信号命中时自动生成关联区域作业票暂停建议。
   - API 模式页面已聚合真实 GDS 点位、活动预警和联动规则，监测模拟写入后同步服务端票证状态，并提供只读刷新核对入口。
   - 待接入 CA 电子签名、GIS 点选、人员定位与移动现场终端。
4. 状态持久化（原型已补齐）
   - 已使用版本化 localStorage 保存告警、隐患、票证、规则、预案和事件生命周期操作，并提供演示数据重置。
   - 风险和隐患域已具备后端数据库模型、并发控制和服务端规则；其余能力域仍需迁移。

### P1：闭环能力增强

1. 预警规则配置（原型已补齐）
   - 已完成服务端草稿、同作用域表达式冲突检测、QHSE/生产负责人异人会签、25%/50%/100% 灰度配置、版本快照、回滚草稿、启停和发布审计。
   - 已完成统一样本执行、AND/OR 表达式、持续时长、联合数据时间窗口、确定性灰度分群、5 分钟去重抑制、预警信号持久化和作业许可暂停建议联动。
   - 前端 API 模式已直接接入规则领域接口和测试样本执行，草稿和运行版本相互隔离。
   - 已接入 MQTT/WebSocket 真实采样链路、区域过滤、断线补传和乱序数据策略。
   - 遥测原始字段已标准化为 GDS/VOC/MES 规则指标，表达式阈值可引用同一样本中的设备限值，避免点位报警但规则不触发。
   - 联合预警按区域汇聚不同点位的 GDS/MES 指标，GDS 趋势由前后样本自动派生，可在配置时间窗口内生成真实联合信号。
   - 已完成预警信号确认、证据核验、应急事件幂等生成、关闭、操作轨迹、区域权限、审计和乐观锁；待补生产 broker TLS 联调。
2. 应急事件全生命周期（原型已补齐）
   - 已完成 NestJS/Prisma 纵向切片、内存与 PostgreSQL 双仓储、告警去重转事件、研判、升级、降级、终止和区域数据范围。
   - 预警详情已接入真实联动入口，同步更新预警状态与应急事件数据源；接口同时要求预警处置和应急管理权限，并支持失败后的幂等重试。
   - 关闭申请已复用服务端通用审批流，具备 4 小时时限、异人审批、证据门槛、意见、电子签名、催办、审计和乐观锁。
   - 前端 API 模式已接入独立事件数据切片和真实证据上传/下载；待对接 CA 签名和真实消息催办。
3. 预案管理流程（原型已补齐）
   - 已完成 NestJS/Prisma 纵向切片、双仓储、新建/独立草稿、QHSE/生产负责人异人会签、不可变版本、历史回滚和乐观锁。
   - 演练计划、启动和评分复盘已接入服务端，前端 API 模式直接使用预案领域接口。
   - 待接入 CA 签名、定期演练自动排程、演练附件和整改任务联动。
4. 资源管理流程
   - 已完成 NestJS/Prisma 纵向切片、内存与 PostgreSQL 双仓储、资源入库、批次与有效期、FEFO 调拨、过期库存拦截、到位归还、巡检维护、权限审计和乐观锁。
   - 前端 API 模式已接入独立资源数据切片，调拨人和巡检人均取当前认证主体。
   - 待补仓库/库位、扫码盘点、维修工单、完整库存流水和跨库调拨。
5. 融合通信真实接口
   - 已完成 NestJS/Prisma 纵向切片、内存与 PostgreSQL 双仓储、0/2/3/5 分钟升级链、送达/失败回执、最多 2 次自动重试、可信确认人、权限审计和乐观锁。
   - 前端 API 模式已接入独立通信数据切片，确认后由服务端阻断继续升级。
   - 待对接电话、短信、App、广播真实网关及号码脱敏。
6. 告警联动证据
   - 已实现事件详情关联监测趋势、工艺参数、同区域作业票、通知人员、证据核验和操作留痕。
   - 四类核验结果已持久化到预警信号，具备区域权限、可信操作人、审计、重复阻断和乐观锁。
   - 已完成隐患与应急事件附件固化、证据哈希、区域权限和服务端审计归档。
   - 待补人员定位和视频快照。
7. 事件调查复盘
   - 已完成独立事件复盘领域、内存/Prisma 双仓储、区域数据范围和复盘台账查询。
   - 应急事件审批关闭后自动幂等创建复盘档案，重复关闭请求可补偿缺失档案且不会重复生成。
   - 已完成事件摘要、直接原因、根本原因和经验教训的结构化维护，调查负责人取认证主体，结论未完整时禁止归档。
   - 已接入统一对象存储，调查报告、现场照片、检测报告和培训记录具备哈希、可信上传人、区域权限和下载能力。
   - 支持新增整改措施，并在措施完成前调整标题、责任部门、责任人、期限和优先级；全部写入继续使用乐观锁。
   - 整改措施按“待整改 → 整改中 → 已完成”受控推进，操作人取认证主体，并使用版本号阻止并发覆盖。
   - 整改措施可幂等转为隐患治理任务，继承标题、责任部门、责任人、期限和措施；风险单元必须与事件区域一致。
   - 已关联措施由隐患领域统一推进，复盘页可同步“待整改/整改中/待验收/已关闭”，隐患关闭后措施自动映射为已完成。
   - 已完成服务端复盘报告导出，报告包含调查结论、全过程时间线、整改/隐患关联、证据目录、版本和导出审计信息，可直接打印为 PDF。
   - 已完成企业知识库结构化检索，统一检索已归档复盘、已关闭隐患和已发布预案，支持中文短语、类型筛选、相关性排序与区域权限。
   - 全部整改完成后才能由具备应急关闭审批权限的人员关闭归档；前端 API 模式使用服务端权威数据，Mock 模式保持独立演示。
   - 待生产化增强附件正文解析、OCR、全文索引和向量语义检索。

### P2：平台化与生产化

1. 组织、用户、角色和数据权限管理。
   - 后端基线已完成组织/区域模型、8 类角色、用户授权、会话认证和 IAM 查询接口。
   - 已增加登录失败限流、单账号活动会话上限、安全响应头和生产 Swagger 默认关闭策略。
   - 已完成内存/Redis 双会话存储，HTTP 和遥测 WebSocket 共用异步认证链，会话故障快速失败。
   - 已完成权限管理页面、账号启停、组织/角色/区域授权、内存/Prisma 双仓储、乐观并发控制和操作审计；角色变化即时作用于现有会话，停用账号立即失效。
   - 待补用户新增、角色自定义、统一身份认证、权限变更审批和多实例授权变更广播。
2. 菜单、按钮、数据范围权限和重要操作二次确认。
   - 风险和隐患模块已接入菜单/按钮权限、后端权限点和区域数据范围强制过滤。
   - 待扩展到其他业务菜单，并补重要操作二次确认。
3. 操作日志、登录日志、告警确认日志和预案启动审计。
   - 已完成登录成功/失败、退出、401/403 安全拒绝、风险评估、措施更新、告警确认/核验/关闭、应急启动和通信操作审计。
   - 待补生产日志平台归集、长期留存策略和异常行为告警。
4. 统计分析、趋势报表、单位绩效和导出能力。
   - 已完成隐患、预警、作业许可、应急事件四领域统一指标、30 天趋势、区域绩效明细和 UTF-8 CSV 导出。
   - 已执行日期范围限制、区域数据权限二次裁剪和报表导出审计；待补自定义指标口径、定时报表和 PDF/Excel 模板。
5. 数据字典、编码映射、接口配置和数据质量监控。
   - 已完成数据字典与非敏感集成登记的内存/Prisma 双仓储、种子、读写权限、写入审计和乐观并发控制。
   - 集成地址执行协议匹配并拒绝内嵌凭据；待补真实连通性探测、数据质量规则和密钥管理系统对接。
6. WebSocket / MQTT 实时数据、断线重连和时间同步。
   - 已完成统一遥测点位/样本模型、幂等写入、质量码、区域数据范围、MQTT QoS 1 适配、WebSocket 权限广播、游标补传、乱序保护、未来时钟偏差校验和前端实时刷新。
   - 已完成遥测字段到预警规则指标的服务端映射及动态上下限解析，GDS 重大预警可真实联动同区域作业票暂停建议。
   - 已完成同区域 GDS/MES 跨点位关联、趋势派生和联合信号回显，MES 页面模拟入口使用催化裂化装置真实关联点位。
   - 待补跨实例持久化事件游标、生产 broker 双向 TLS、连接监控告警和容量压测。
7. 运行诊断和可观测性。
   - 已完成服务/存储模式、进程内存、集成状态和路由级请求量、错误率、延迟聚合，并提供 15 秒自动刷新的受保护诊断页。

   - 已完成 W3C 上下文传播、真实服务端 span 生命周期和可选 OTLP/HTTP 批量导出，API 响应与结构化日志统一携带 `traceId`。
   - 动态路径按路由模板聚合，接口不返回集成地址或凭据；待接入 Prometheus、集中日志、Collector 生产部署和多实例告警。
8. 异常边界、错误恢复、无障碍、移动端和值守大屏长时间运行测试。

9. 后端、数据库、缓存、消息队列、统一认证和部署运维体系。
