## 需要完善的功能和模块

### P0：核心业务补齐

1. 风险分级管控（原型已补齐）
   - 已实现风险单元、动态因子、管控措施和 GDS/VOC/MES/隐患/票证关联展示。
   - 待补风险辨识评估表单、管控措施维护和组织级风险单元树。
2. 隐患排查治理（原型已补齐）
   - 已实现分级台账、逾期标识和整改、验收、关闭状态流转。
   - 待补上报表单、整改证据上传、挂牌督办和重复隐患统计。
3. 作业许可管理（原型已补齐）
   - 已覆盖动火、受限空间、高处、吊装、临时用电，并实现同区域告警暂停、复测恢复。
   - 待补完整审批流、电子签名、空间距离计算和现场终端确认。
4. 状态持久化（原型已补齐）
   - 已使用版本化 localStorage 保存告警、隐患、票证和规则操作，并提供演示数据重置。
   - 正式系统仍需接入后端数据库、并发控制和事件状态机。

### P1：闭环能力增强

1. 预警规则配置（原型已补齐）
   - 已实现规则查询、启停、阈值/持续时间/范围/通知链展示，并接入模拟告警场景。
   - 待补规则新增编辑、表达式校验、审批发布和版本回滚。
2. 应急事件全生命周期
   - 补齐研判、升级、降级、终止、关闭审批和操作留痕。
3. 预案管理流程
   - 新建、编辑、评审、发布、版本对比、到期提醒和演练记录。
4. 资源管理流程
   - 入库、领用、盘点、检查、维护、过期提醒和调度轨迹。
5. 融合通信真实接口
   - 对接电话、短信、App、广播平台，处理重试、失败回执和号码脱敏。
6. 告警联动证据
   - 事件详情统一展示工艺趋势、周边测点、作业票、人员和视频快照。

### P2：平台化与生产化

1. 组织、用户、角色和数据权限管理。
2. 菜单、按钮、数据范围权限和重要操作二次确认。
3. 操作日志、登录日志、告警确认日志和预案启动审计。
4. 统计分析、趋势报表、单位绩效和导出能力。
5. 数据字典、编码映射、接口配置和数据质量监控。
6. WebSocket / MQTT 实时数据、断线重连和时间同步。
7. 异常边界、错误恢复、无障碍、移动端和值守大屏长时间运行测试。
8. 后端、数据库、缓存、消息队列、统一认证和部署运维体系。

## 建议迭代顺序

1. 风险分级管控 + 隐患治理 + 作业许可，完成 QHSE 管理主业务。
2. 事件状态机 + 规则编辑发布，使现有演示闭环具备正式流程和版本管理。
3. 权限审计 + 统计报表 + 基础配置，形成可交付的管理平台。
4. 真实数据和通信接口接入，最后开展性能、安全和生产部署改造。
